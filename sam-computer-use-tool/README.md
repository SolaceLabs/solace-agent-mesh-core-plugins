# sam-computer-use-tool

A browser automation tool for Solace Agent Mesh using Playwright. Enables LLM agents to control web browsers, navigate pages, click elements, type text, and capture screenshots.

## Features

- **Browser Automation**: Full browser control via Playwright (Chromium, Firefox, WebKit)
- **Normalized Coordinates**: LLM-friendly 0.0-1.0 coordinate system
- **Artifact Storage**: Screenshots saved to SAM's artifact storage (not inline base64)
- **LLM-Generated Filenames**: Descriptive screenshot names generated by the LLM
- **On-Demand Browser**: Browser starts only when `open_browser` is called

## Installation

```bash
solace-agent-mesh plugin add computer-use-agent --plugin sam-computer-use-tool
```

After installation, install Playwright browsers:

```bash
playwright install chromium
```

## Configuration

### Tool Configuration

```yaml
tools:
  - tool_type: python
    component_module: "sam_computer_use_tool.tools"
    class_name: "ComputerUseTool"
    tool_config:
      tool_name: "computer_use"
      viewport:
        width: 1920
        height: 1080
      browser:
        headless: true
        browser_type: chromium  # chromium, firefox, or webkit
        timeout_ms: 30000
      normalize_coordinates: true
      screenshot_format: png
      use_llm_naming: true
```

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `BROWSER_HEADLESS` | Run browser in headless mode | `true` |
| `BROWSER_TYPE` | Browser engine | `chromium` |
| `BROWSER_VIEWPORT_WIDTH` | Viewport width | `1920` |
| `BROWSER_VIEWPORT_HEIGHT` | Viewport height | `1080` |

## Usage

### Actions

| Action | Parameters | Description |
|--------|------------|-------------|
| `open_browser` | `url` (optional) | Start browser, optionally navigate to URL |
| `close_browser` | - | Close browser and cleanup |
| `navigate` | `url` | Navigate to URL |
| `click` | `x`, `y`, `button` | Click at coordinates |
| `type` | `x`, `y`, `text`, `press_enter` | Type text at coordinates |
| `scroll` | `direction`, `amount`, `x`, `y` | Scroll page or at position |
| `hover` | `x`, `y` | Move mouse to position |
| `wait` | `duration_ms` | Wait for milliseconds |
| `go_back` | - | Browser back |
| `go_forward` | - | Browser forward |
| `search` | `query` | Google search |
| `key_combination` | `keys` | Press keyboard shortcut |
| `drag_and_drop` | `x`, `y`, `end_x`, `end_y` | Drag element |
| `screenshot` | - | Capture screenshot only |

### Coordinate System

Coordinates are normalized between 0.0 and 1.0:
- `(0.0, 0.0)` = top-left corner
- `(1.0, 1.0)` = bottom-right corner
- `(0.5, 0.5)` = center of viewport

### Example Workflow

```
User: Go to google.com and search for "Solace PubSub+"

Agent:
1. computer_use(action="open_browser", url="https://google.com")
2. describe_image(filename="google_homepage_143522.png")  # Understand page
3. computer_use(action="type", x=0.5, y=0.4, text="Solace PubSub+", press_enter=true)
4. describe_image(filename="google_search_results_143535.png")  # See results
5. computer_use(action="close_browser")
```

## Response Format

Each action returns:

```json
{
  "status": "success",
  "action": "navigate",
  "current_url": "https://example.com",
  "page_title": "Example Domain",
  "viewport_size": [1920, 1080],
  "screenshot": {
    "filename": "example_domain_homepage_143522.png",
    "version": 0,
    "mime_type": "image/png"
  }
}
```

## Integration with describe_image

The tool works best with SAM's built-in `describe_image` tool. After each action, use `describe_image` to analyze the screenshot and understand the page content before deciding the next action.

```yaml
tools:
  - tool_type: builtin
    tool_name: "describe_image"
    tool_config:
      <<: *image_description_model
```
